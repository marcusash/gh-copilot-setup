Step 1: Download and extract

Download your setup package: Download gh-copilot-setup.zip

Save it anywhere (Desktop or Downloads is fine), then right-click and choose Extract All.

Windows SmartScreen warning: this is expected.

Because the scripts are downloaded from the internet and not code-signed, Windows will show a blue "Windows protected your PC" dialog when you run setup.bat. This is normal.

Click More info, then click Run anyway to continue.

Step 2: Run the installer

Inside the extracted folder, double-click setup.bat. It will request administrator access and launch the installer automatically.

The installer will ask for your name, GitHub username, and email. Enter your details and let it run. It takes about 10 minutes on a fresh machine.

If a browser window opens during install (for GitHub or WorkIQ login), sign in and return to the PowerShell window. The installer will continue automatically.

Step 3: Close and reopen Windows Terminal

After the installer finishes and you see the green "Setup Complete!" banner, close PowerShell and reopen Windows Terminal. Fonts and themes take effect after a restart.

The first time you open AI Maker, it will run an onboarding interview. This takes about 5 minutes. It asks you 10 questions about how you work, what you need help with, and how you prefer to communicate.

Your answers are saved to C:\AIMaker\profile.md. AI Maker reads this file at the start of every session so you never have to re-explain yourself.

Answer honestly and specifically. The more detail you give, the better AI Maker will adapt to you from day one.

After the interview

AI Maker will greet you by name from session two onward. It knows your preferences, your biggest time drains, and how you want feedback delivered. You can update your profile any time: "Update my profile" or "Remember that I..."

Tutorial 1 of 4

My GitHub Home

Learn the basics of GitHub by building your own presence. You will create a profile README, make your first commit, and understand how GitHub organizes work. This is the foundation for everything else in this environment.

You will use: AI Workbench

What you will build: A GitHub profile README that introduces who you are and what you work on.

Part 1: Understand what GitHub is

GitHub is where code and work lives. Everything in your AI environment is stored in GitHub repos. Your agents read from GitHub. Your pc-setup lives there. Understanding how it works lets you take control of your own environment.

Key concepts

  • Repository (repo): A folder on GitHub that tracks every change ever made to the files inside it.
  • Commit: A saved snapshot of changes. Every commit has a message explaining what changed and why.
  • Branch: A copy of the repo where you can make changes without affecting the main version.
  • Pull request (PR): A proposal to merge changes from one branch into another. The place where code review happens.
  • Issue: A task, bug, or discussion item tracked inside a repo.

Your pc-setup repo is already on GitHub at github.com/YOUR_GITHUB_USERNAME/pc-setup. Every time you improve your setup and push the changes, your future self (or a new machine) benefits from the update.

Part 2: Create your GitHub profile README

GitHub has a special repo: if you create a repo with the same name as your username, its README appears on your GitHub profile page. This is your GitHub home page.

1

Open AI Workbench

Click the red "AI Workbench" tab in Windows Terminal.

2

Create the profile repo

Type this in AI Workbench (replace YOUR_GITHUB_USERNAME with your actual username):

gh repo create YOUR_GITHUB_USERNAME/YOUR_GITHUB_USERNAME --public --description "My GitHub profile"

Then clone it locally:

git clone https://github.com/YOUR_GITHUB_USERNAME/YOUR_GITHUB_USERNAME C:\GitHub\my-profile
cd C:\GitHub\my-profile
3

Ask AI Workbench to draft your README

Tell AI Workbench:

"Write a GitHub profile README for me. I'm [your name], [your role] at Microsoft. I work on [what you work on]. Keep it to 3-4 sentences, professional but not stiff."

AI Workbench will generate the README content. Review it. Ask for changes if anything doesn't sound right.

4

Save and commit it

Tell AI Workbench:

"Save this as README.md in C:\GitHub\my-profile, then commit it with a message explaining it's my first profile README."

AI Workbench will write the file and run the git commands for you. The commands it runs will look like:

git add README.md
git commit -m "Add profile README"
git push origin main
5

See it on GitHub

Open your browser and go to github.com/YOUR_GITHUB_USERNAME. Your README should appear on your profile page.

Part 3: Understand issues and how work gets tracked

Issues are how work gets tracked on GitHub. They are like tasks or notes attached to a repo. Every improvement you want to make to your setup can start as an issue.

Try it

Tell AI Workbench:

"Create 3 issues in my pc-setup repo: one for adding a new skill, one for updating my README, and one idea I have for improving my setup. Use gh to create them."

AI Workbench will create the issues using gh issue create. Then open GitHub in your browser and find them in the Issues tab of your pc-setup repo.

What to take away: Every change you want to make to your environment can live as an issue first. Issues give you a place to collect ideas without committing to them yet.

What you learned

  • What GitHub repos, commits, branches, and issues are
  • How to use AI Workbench to run GitHub commands without memorizing them
  • How your pc-setup repo connects to your live machine
  • How to create and track work as GitHub issues

Tutorial 2 of 4

Meeting Intelligence Report

Use AI Workbench's researcher skill and WorkIQ to analyze your calendar and build a weekly meeting summary. This teaches you how skills work, how WorkIQ connects to your M365 data, and how to turn raw data into a useful report.

You will use: AI Workbench

What you will build: A weekly meeting intelligence report you can actually use on Monday morning.

Part 1: Connect WorkIQ (one-time setup)

WorkIQ is the connection between your agents and your Microsoft 365 account. Once authenticated, both agents can access your meetings, emails, Teams conversations, and recent documents.

In AI Workbench, type:

"Authenticate WorkIQ for me."

AI Workbench will open a browser window with a device code login. Sign in with your Microsoft account. After signing in, close the browser and come back to AI Workbench. WorkIQ is now connected.

WorkIQ authentication lasts for your session. The next time you launch AI Workbench, you may need to re-authenticate. This is normal. It takes 10 seconds.

Part 2: Pull your meeting data

Tell AI Workbench:

"Use WorkIQ to pull my meetings from last week. Show me a list: meeting name, duration, attendees, and whether I organized it or was invited."

AI Workbench will use the WorkIQ skill to query your calendar and return a structured list. Review it. If anything looks wrong (meetings missing, wrong names), tell AI Workbench what you expect to see.

What to look for

  • How many meetings did you have total?
  • How many did you organize vs attend?
  • Which meetings were recurring vs one-time?
  • How many hours total?

Part 3: Run the researcher skill

The researcher skill teaches AI Workbench to produce findings with stated confidence levels, not just summaries. Tell AI Workbench:

"Analyze my meeting list with the researcher skill. I want to know: what patterns do you see in how I spend my meeting time? Are there meetings I could consolidate or drop? Give me your findings with confidence levels."

AI Workbench will produce a structured analysis. Each finding will have:

  • What the data shows (observation)
  • What it infers from that (conclusion)
  • A confidence level (CONFIRMED / HIGH / MEDIUM / LOW)
  • What would make the finding misleading

If a finding has a LOW confidence level, ask AI Workbench what additional data it would need to raise confidence. This is how you learn to use the researcher skill well.

Part 4: Build the report

Tell AI Workbench:

"Turn this analysis into a meeting intelligence report I can use every Monday. Format it as: a one-paragraph summary, a data table, three findings with confidence levels, and one recommendation for next week. Save it as a canvas."

AI Workbench will generate an HTML canvas file and tell you its path (something like C:\AIWorkbench\canvas\2026-03-meeting-report.html). Open it in your browser.

Make it a weekly habit

Tell AI Workbench to save the template to your vault:

"Save the meeting report format as a how-to in my vault so I can run it every Monday."

AI Workbench will write a step-by-step guide to C:\AIWorkbench\vault\how-to\weekly-meeting-report.md. From now on, running your weekly meeting intelligence report is a one-sentence request.

What you learned

  • How WorkIQ connects your agents to your M365 data
  • How the researcher skill structures analysis with confidence levels
  • How to turn a WorkIQ data pull into a usable report
  • How to use the vault to save processes you want to repeat

Tutorial 3 of 4

Team Onboarding Kit

Use both agents together to build a real onboarding document for a new team member. AI Maker handles the writing and voice. AI Workbench handles research, fact-checking, and structure. This tutorial shows how the two agents work as a team.

You will use: AI Maker and AI Workbench

What you will build: A working onboarding doc you can actually hand to a new hire.

Part 1: Research phase (AI Workbench)

Before writing anything, you need to gather the raw material. Start in AI Workbench.

1

Pull team context from M365

"Use WorkIQ to find the last 3 team announcements or all-hands emails I sent. Also find recent documents I've shared with my team. I want raw material for an onboarding doc."
2

Extract the key information

"From what you found: what are the 5 most important things someone new to my team needs to know? Use the researcher skill. Give me findings with confidence levels."
3

Save the research

"Save this research as a reference in my vault: 'team-onboarding-research.md'. Include the raw findings and confidence levels so I can check them."

Part 2: Writing phase (AI Maker)

Switch to the AI Maker tab. Now use the research to write the actual document.

1

Paste in the research

Open C:\AIWorkbench\vault\references\team-onboarding-research.md and copy the findings. Paste them into AI Maker:

"Here's research AI Workbench pulled about my team. [paste findings]. Use this to write an onboarding doc for someone joining my team. 3-5 pages. Real content, no filler. Sound like me."
2

Review and refine

Read the draft. Tell AI Maker what to change. This might take 2-3 rounds. Examples:

"The section on team values sounds too formal. Rewrite it in my voice."
"Add a section on how we run our weekly standup."
"The intro is good. The middle section needs a table instead of paragraphs."
3

Put it on canvas

"Put the final version on the canvas as a polished document I could share with a new hire."

AI Maker will generate an HTML canvas file. Open it in your browser. This is the deliverable.

Part 3: Cross-check (AI Workbench)

Switch back to AI Workbench for a quick quality check.

"Review this onboarding doc draft: [paste the text]. Is anything factually inconsistent with the M365 data you pulled earlier? Flag anything that looks like it was made up rather than sourced from real data."

AI Workbench will compare the draft against the original source data. If it flags anything, go back to AI Maker to correct it.

The key habit: AI Maker writes. AI Workbench verifies. Using both agents in sequence is how you get output that is both well-written and accurate.

What you learned

  • How to use the two agents together: research first, write second, verify third
  • How to pass context from AI Workbench to AI Maker
  • How to use the canvas to produce a shareable document
  • How the researcher skill adds a quality check to AI-generated writing

Tutorial 4 of 4

Your Voice Skill

Build a skill file that teaches AI Maker how you write. After this tutorial, every email AI Maker drafts will sound like you: your tone, your sentence length, your way of opening and closing, the words you use and the ones you never use.

You will use: AI Maker

What you will build: A voice skill file at C:\AIMaker\.github\skills\my-voice.md

What a skill is

A skill is a text file that gives your agent specialized knowledge it does not have by default.

Think of it like this: when you hired someone new, you probably told them a few things on day one. "We're pretty direct on this team." "We don't do a lot of formal back-and-forth." "When you send the team a status update, keep it to three bullet points." That's a skill. You are encoding the rules of your world so your agent doesn't have to guess.

AI Maker reads your skills every time it starts. The more specific your skills are, the better it performs.

Your skills live in one folder on your machine:

C:\AIMaker\.github\skills\

AI Maker currently has 11 built-in skills: research, design, code, quality, operations, data, brainstorming, user research, canvas, vault, and WorkIQ. You are about to add a 12th: your voice.

How to install a skill from anywhere on the internet

Skills are just text files. Anyone can publish a skill as a GitHub repo, and you can install it in three steps.

Example: the Frontend Slides skill
This skill (github.com/zarazhangrui/frontend-slides) teaches an agent to build beautiful HTML presentations from scratch. To install it for AI Workbench:

Step 1: Open AI Workbench and run:

git clone https://github.com/zarazhangrui/frontend-slides C:\Temp\frontend-slides-skill

Step 2: Copy the skill file into your agent's skills folder:

Copy-Item C:\Temp\frontend-slides-skill\SKILL.md C:\AIWorkbench\.github\skills\frontend-slides.md

Step 3: Clean up and restart:

Remove-Item C:\Temp\frontend-slides-skill -Recurse -Force

Then close and reopen AI Workbench. It will load the new skill automatically.

That's the entire install process. Any skill anyone has published on GitHub can be installed this way. Search GitHub for topic:copilot-skill or topic:claude-skill to discover skills others have built.

Building your voice skill

This tutorial has four phases. You will do all of them inside AI Maker. Open AI Maker now.

Phase 1

Pull your emails with WorkIQ

AI Maker will use WorkIQ to look at emails you have actually sent. This is the raw material for your voice profile.

Type this in AI Maker:

"I want to build my voice skill. Use WorkIQ to pull the last 20 emails I sent to my team. Show me the subjects and a summary of each one."

AI Maker will use WorkIQ to connect to your Outlook and retrieve your recent sent mail.

What to look for

AI Maker will show you a list. Scan it. If any emails are confidential, one-off, or not representative of how you normally write, tell AI Maker to remove them from the sample. You want emails that sound like the real you, not emails you wrote in a rush or ones that were unusually formal.

When the sample looks right, type:

"Now analyze these emails for my voice patterns. What do you notice about how I write? Look at: sentence length, how I open and close, how formal I am, how I handle asks and next steps, words I use often, and things I never say."

AI Maker will return an analysis. This is your voice fingerprint.

Phase 2

The voice interview

After AI Maker shows you the analysis, it will ask you 6 questions to fill in the gaps. Answer honestly. The more specific you are, the more accurate your skill will be.

QuestionWhy it matters
"Does this analysis sound right to you?"You know your own voice better than any tool. Correct anything that's off.
"Share an email you felt really good about."This becomes your anchor example. AI Maker calibrates everything against it.
"Share an email where the tone was off."Knowing what to avoid is as important as knowing what works.
"How do you handle hard messages?"Hard messages are where voice breaks down. Getting this right is where the skill earns its value.
"Three phrases you never want to see in my drafts."Everyone has a list. Write yours down. One instruction prevents it forever.
"Does this test draft sound like you?"AI Maker writes a sample email. You score it and explain what felt off.
Phase 3

Build and install the skill file

After the interview, type:

"Now write my voice skill file. Use everything we found: the email analysis, my answers, the anchor example, and the corrections I made to the test draft."

AI Maker will write a skill file. It will look something like this (filled in with your actual patterns):

# My Email Voice

## Who this is for
[Your name]. [Your role]. Sending team emails.

## Tone
[Direct and warm. Team-first. We over I.]

## Sentence length
[Short to medium. Rarely more than 2-3 sentences per paragraph.]

## How I open emails
[Get to the point in the first sentence. No "I hope this finds you well."]

## How I handle asks
[One clear ask per email. Never bury the ask at the end.]

## How I close
[Action-oriented. Never "please don't hesitate."]

## Words I use
[we, let's, here's what I'm thinking, the goal is, quick update]

## Words I never use
[circling back, per my last email, touching base, leverage]

## Hard messages
[Direct but not harsh. Give the context, state the issue, offer a path forward.]

## Anchor email (this is what "sounds like me" looks like)
[The full text of your best example email]

To install it, type:

"Save this as my-voice.md in C:\AIMaker\.github\skills\"

AI Maker will write the file directly to your skills folder. Close and reopen AI Maker, and your voice skill is live.

Phase 4

Test and tune

Ask AI Maker to write a real email for you:

"Using my voice skill, draft an email to my team about [your actual topic]."

Read the draft. Rate it:

  • Does it sound like you?
  • Would you send it as-is, or would you change something?
  • Is there a word or phrase that feels off?

If something is wrong, just tell AI Maker. You never need to open any files.

"That sign-off doesn't sound like me. Update my voice skill so you never end an email that way."
"I would never say 'please don't hesitate.' Add it to my never-use list."
"This draft is too formal. Adjust my voice skill: my default tone is warmer than this."

AI Maker will update C:\AIMaker\.github\skills\my-voice.md directly and apply the change going forward. You refine your voice skill by talking to AI Maker, not by editing text files.

This is normal. Voice skills improve with iteration. Most people do 2-3 correction conversations before the output feels natural. Your anchor email is your reference point every time.

Tuning your skill over time

Your voice skill is a living document. You tune it by talking to AI Maker, not by editing files.

  • Add a new anchor email whenever you write something you love: "That email was perfect. Add it to my voice skill as a new example."
  • Add to your never-use list whenever AI Maker generates a phrase that bothers you: "Never write 'synergy' in my emails."
  • Add a new scenario when you have a new type of email to write: "Add a section to my voice skill for how I write executive status updates."
