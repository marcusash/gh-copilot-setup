Your Two Agents
AI Maker (Yellow tab): Your day-to-day partner. Emails, documents, brainstorming, presentations, meeting prep. Use AI Maker for work you already do.
AI Workbench (Red tab): Your technical partner. GitHub management, PowerShell scripts, data analysis, research with citations, debugging. Use AI Workbench for technical depth.
When in doubt, start with AI Maker. It will hand off to AI Workbench if the task needs technical depth.
Tutorial 1: My GitHub Home
TUTORIAL 1 OF 4 • USE AI WORKBENCH
What you will build
A GitHub profile README that introduces who you are. Learn how repos, commits, branches, and issues work.
1
Create your profile repo
Tell AI Workbench:
"Create a GitHub repo with my username as the repo name. Make it public. This is my profile repo."
AI Workbench will run gh repo create for you.
2
Draft your README
Tell AI Workbench:
"Write a GitHub profile README for me. I'm [your name], [your role]. I work on [what you work on]. Keep it to 3-4 sentences."
3
Commit and push it
Tell AI Workbench:
"Save this as README.md in the profile repo, commit it, and push to GitHub."
AI Workbench will run the git commands for you.
What you learned: How to use AI Workbench to run GitHub commands without memorizing them. How repos and commits work.
Tutorial 2: Meeting Intelligence
TUTORIAL 2 OF 4 • USE AI WORKBENCH + WORKIQ
What you will build
A weekly meeting intelligence report. Learn how WorkIQ connects to your M365 data and how to analyze meetings.
1
Connect WorkIQ (one-time setup)
Tell AI Workbench:
"Authenticate WorkIQ for me."
A browser window will open. Sign in with your Microsoft account. Close the browser when done.
2
Pull your meeting data
Tell AI Workbench:
"Get all my meetings from the last 7 days. Pull the title, attendees, and duration for each one."
AI Workbench will query WorkIQ and return the data.
3
Analyze and summarize
Tell AI Workbench:
"Summarize this week's meetings. Group by project. Flag any meetings where I was the only attendee from my team."
What you learned: How WorkIQ connects your agents to M365 data. How to query and analyze your calendar without leaving the CLI.
Tutorial 3: Team Onboarding Kit
TUTORIAL 3 OF 4 • USE AI MAKER
What you will build
An onboarding guide for new teammates. Learn how AI Maker handles structured documents and brainstorming.
1
Brainstorm the content
Tell AI Maker:
"I need to create an onboarding guide for new engineers on my team. What should be in it? Ask me clarifying questions first."
AI Maker will interview you before generating content.
2
Generate the guide
Tell AI Maker:
"Based on what I just told you, write the onboarding guide. Use markdown. Keep it to one page."
3
Refine it
If the tone is off, tell AI Maker:
"This is too formal. Rewrite it like I'm talking to a peer on their first day."
AI Maker will remember this preference for future documents.
What you learned: How AI Maker asks clarifying questions before generating content. How to refine tone and structure through iteration.
Tutorial 4: Your Voice Skill
TUTORIAL 4 OF 4 • USE AI WORKBENCH
What you will build
A custom skill that teaches your agents how you communicate. Learn how to extend your agents with your own instructions.
1
Create the skill file
Tell AI Workbench:
"Create a new skill file called 'my-voice.md' in my pc-setup repo. This skill will teach you how I write emails and documents."
2
Write the skill instructions
Tell AI Workbench:
"Add these rules to the skill: I prefer bullet points over paragraphs. I use short sentences. I avoid corporate jargon. I sign emails with just my first name."
3
Test the skill
Switch to AI Maker. Tell it:
"Draft an email to my team about the Q3 roadmap delay. Use my voice skill."
AI Maker will apply the rules you just defined.
What you learned: How to create custom skills. How skills extend what your agents know. How to maintain your own agent instructions in GitHub.
