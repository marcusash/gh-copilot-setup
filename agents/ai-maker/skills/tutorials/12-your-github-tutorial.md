# Teach your agent to run your project with you every day

**What this unlocks**
Tutorial 1 taught you what GitHub is. This skill is different: it is a working habit. After you tell your agent about your repo, it helps you run it every session. What is open this week? What changed since Friday? Help me close this issue. Draft a commit message for what I just shipped. Your agent becomes the partner for your actual project, not just a tool you used to set things up once.

**Try it now**
- "What issues are open in my repo right now? Summarize what needs attention."
- "I just finished this task. Help me write a closing comment and close the issue."
- "What changed in my repo since last Friday?"

**What to ask after**
- "What should I work on next based on what's open?"
- "Draft a README for what I'm building. Here's the one-sentence description: [your description]."
- "Help me write a commit message for what I just did."
- "I have three open issues. Which is the highest priority given what I'm trying to ship?"
- "Look at this week's commits. Write me a short summary of what shipped."

**Tune it to your world**
Tell your agent your repo URL, what you are building in one sentence, and how you track work (labels, milestones, project boards). Tell it your commit style and your release cadence. It applies your conventions every session without you having to repeat them.

> **This skill is for red pill users.** Blue pill users: complete Tutorial 1 (My GitHub Home) first to create your repo, then come back here.
