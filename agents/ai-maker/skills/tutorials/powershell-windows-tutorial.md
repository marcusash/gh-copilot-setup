# Teach your agent to work in your Windows terminal like it belongs there

**What this unlocks**
Your agent can already suggest PowerShell commands. This skill teaches it to do it safely: reading before writing, previewing before running, using native cmdlets that actually work in scripts, and quoting every path so nothing breaks on a machine with a space in a folder name. When something goes wrong at the terminal, you stop debugging together by guessing and start by reading the actual error.

**Try it now**
- "Write me a PowerShell script that does this: [describe what you want]. Tell me what it will do before you run it."
- "This is the error I got: [paste error]. What should I check first?"
- "I want to automate this task on my Windows machine. Where do I start?"

**What to ask after**
- "Check this script for anything that could break on a different machine."
- "I need to stop a process that's hanging. How do I do it safely?"
- "Write a script that runs this every morning. Make it safe to run repeatedly."
- "Why is this command working in the terminal but failing in the script?"
- "I want to read a settings file and modify one value. Write that safely."

**Tune it to your world**
Tell your agent which tools you use most (winget, git, Node, Python, VS Code) and which folders matter on your machine. It will tailor its scripts to your environment instead of writing for a generic Windows install.
