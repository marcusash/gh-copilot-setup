A complete Windows developer environment setup with two personalized GitHub Copilot agents: AI Maker and AI Workbench. One install script sets up your whole machine: Git, GitHub CLI, VS Code, Windows Terminal, and both agents.

> DETAILED INSTALLATION STEPS
1
Get a GitHub account
If you don't have one: github.com/signup
2
Subscribe to GitHub Copilot Pro
$10/month minimum. The free plan does not support custom agents or CLI access.
Subscribe: github.com/features/copilot/plans
3
Open Windows Terminal as Administrator
Press Win+X and select "Terminal (Admin)".
Or: search "Terminal" in Start menu, right-click, "Run as administrator".
4
Run the setup script
irm https://raw.githubusercontent.com/marcusash/gh-copilot-setup/main/install.ps1 | iex
The script will:
  • Ask for your name and GitHub username
  • Install Git, GitHub CLI, Node.js, VS Code, PowerShell 7, Windows Terminal
  • Create a GitHub repo (your-username/pc-setup) and push your setup files to it
  • Install AI Maker and AI Workbench, personalized with your name
  • Optionally set up WorkIQ (Microsoft 365 integration)
5
Sign in to GitHub
The setup script will open a browser window for GitHub authentication. Follow the instructions. This is a one-time step.
6
Close and reopen Windows Terminal
After the installer finishes with the green "Setup Complete!" banner, close Terminal and reopen it. Fonts and themes take effect after restart.
7
Launch your agents from desktop shortcuts
Double-click AI Agents on your Desktop. Two GitHub CLI tabs open: a yellow tab (AI Maker) and a red tab (AI Workbench).
8
Initialize AI Maker first
In the yellow tab, say: Good morning.
AI Maker will run the onboarding interview. Answer one question at a time. This teaches AI Maker your working style.
9
Initialize AI Workbench
In the red tab, say: AI Workbench ready. What are we working on?
No onboarding interview. It learns through the work.
10
Choose your agent based on the work
Use AI Maker for day-to-day work. Use AI Workbench for technical investigation, research, debugging, or building. Each has its own CLI workspace. Each specializes.
> WHAT YOU GET
AI Maker : design, code, research, quality, data, brainstorming, user research skills. Learns your working style in the first session.
AI Workbench : deep research, debugging, PowerShell and Windows expertise, GitHub management. Built for technical deep dives.
Your own repo : your-username/pc-setup on GitHub. Add your own skills, update agent instructions, and push your changes. This is your environment now.
WorkIQ integration : if you have M365 Copilot, both agents can query your calendar, email, and Teams directly.
> TROUBLESHOOTING
Script execution error
If you see "running scripts is disabled", run this first:
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
Windows SmartScreen warning
Because the scripts are downloaded from the internet and not code-signed, Windows will show a blue "Windows protected your PC" dialog. This is normal.
Click More info, then click Run anyway to continue.
GitHub CLI not found
The setup script should install GitHub CLI automatically. If you see this error, manually install from:
cli.github.com
